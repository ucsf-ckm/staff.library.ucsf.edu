Database Redux

July 5, 2016June 3, 2016 by Michele Mizejewski

As some of you may be aware, the Web Projects Team is working to move the current library website off its old Drupal 6 platform and in the process, also improve user experience. One of the most important elements of the Library site is access to databases, so starting next week the databases page will have a new home. Following the example of many other academic libraries, we will be using the assets part of Springhare’s LibGuides. One benefit is that this content will immediately be more usable on mobile devices.

On the main page, users can begin by choosing a subject or a database content type from a list. They can also just enter a term to search.

Each listing has a title that links out to the actual database, plus a short description and a “more…” link that drops open to show a longer description when necessary. No more opening another page to find details on a database!

Some types of content now get visual icons too, making it plainer to someone skimming a list of databases generated by a search or subject choice.

Note that we will also have the ability to mark resources as “New” or “Trial,” and those will be highlighted until that designation is removed.

One of the best features is the integration and expert input that can be seen when viewing a subject. See Dentistry for a live example. If the subject has an expert assigned or any related guides, those display in the right column. The main content on the left lists all databases tagged with the subject “dentistry,” and the expert can also hand pick a few to be designated as Best Bets.

Bonus: the backend where databases are entered and edited by Collections is much simpler, and EZproxy integration is better.

If you are a subject expert and think that something in Databases needs editing, let Collections or the Web Team know via a ticket. The orange Help button is the quickest way to open one.
